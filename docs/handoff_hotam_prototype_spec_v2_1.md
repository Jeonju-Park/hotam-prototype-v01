## 0. IA v0.1.1 동기화 (기능 추가 전에 먼저)

정주가 xlsx에서 61행의 텍스트를 수정(행 추가/삭제·마일스톤 변경 없음). 패턴: ①진입→연결을 `S4(식당 상세)` 형식으로 ID+이름 병기 ②내부 용어('캐논' 등) 제거 ③비고 정리 ④소통성 메모 5건 신규(diff 리포트 `ia_diff_v0_1_1.md` 참조).

**동기화 스크립트(`scripts/sync_ia.js`) 요구사항:**
1. `ia_inspector_data_v2_1.js`와 Supabase 현재 값을 필드 단위 비교 → **달라진 필드만 update**
2. 모든 update는 `ia_change_log`에 기록 — author `'기획자(정주)'`, 필드·이전값·새값
3. 실행 전 변경 예정 목록을 콘솔에 출력하고 확인 후 진행(dry-run 기본)
4. Supabase에서 사람이 직접 고친 값과 충돌하면(양쪽 모두 v0.1 원본과 다름) 덮어쓰지 말고 충돌 목록으로 출력 — 사람이 결정
5. 완료 후 `src/data/ia_inspector.js`(오프라인 폴백)를 v2_1 파일로 교체

## 1. IA Live에 화면 정보 표시 (요구 1)

- 테이블 상단(기능 행들 위)에 **현재 화면 카드** 상시 표시: 화면ID 뱃지 + 화면명 + Type 칩(Page/Sheet/Tab…) + 마일스톤 뱃지 + 화면 목적. 카드도 클릭 시 아코디언(카피·이슈·비고·메모)으로 확장, 화면 자체도 편집·메모 대상(SCREENS 테이블).
- **브레드크럼 UI**: 카드 최상단에 `영역 › 화면` (기능 아코디언 안에서는 `영역 › 화면 › 기능명`) — 구분/뎁스를 브레드크럼으로 표현. 각 조각은 클릭 가능: 영역 클릭=전체 보기 필터를 그 영역으로, 화면 클릭=해당 화면으로 프로토타입 이동.
- **진입→연결 표기**: 데이터가 이미 `S4(식당 상세)` 병기 형식(v0.1.1) — 셀 렌더 시 `S4`·`S18`·`HTM-…` 패턴을 파싱해 **클릭 가능한 칩**으로: 화면ID 칩 클릭=프로토타입 그 화면으로 이동, 기능ID 칩 클릭=그 행으로 점프+하이라이트. 편집 시에도 병기 형식을 placeholder로 안내.

## 2. 메모 시스템 고도화 — 태그·대상·댓글 (요구 2)

**스키마 변경:**
```
ia_memos     — tag(text: '이슈'|'QA'|'기능'|'화면') 컬럼 추가,
               target_id(기존 — 화면ID/기능ID/null), resolved(bool, 기본 false) 추가
ia_memo_comments — id(uuid PK), memo_id(FK ia_memos), author, body, created_at   ← 신규 테이블
```

**시딩:** `OPEN_ISSUES` 8건(O9~O16)을 메모로 1회 등록 — tag='이슈', body=내용+결정 주체+메모, target_id=관련 화면(첫 번째 ID), author='기획자(정주)'. 추가로 diff 리포트의 소통성 메모 5건(HTM-S13-01·S14-04·S7-10·S7-14·S9-05 비고)을 tag='기능' 메모로 등록.

**메모 탭 UI:**
- 각 메모: 태그 칩(이슈=danger-bg / QA=info-bg / 기능·화면=grade-soso-bg) + **대상 칩 "S7 · 피드" 또는 "HTM-S7-10 · 카드—본문"(ID+이름 병기)** + 작성자·시간 + 본문 + **[바로가기] 버튼**(해당 화면으로 프로토타입 이동 + 대상 요소/행 하이라이트, 화면 태그면 화면 이동만)
- **댓글:** 메모마다 접이식 댓글 스레드(작성자·시간·본문, 입력창) — `ia_memo_comments`. 새 댓글도 실시간 반영+뱃지. 이슈 위에서 대화가 이어지는 구조.
- **메모 추가 버튼(＋)**: 폼 = 태그 선택(4종) → 대상 선택(화면/기능 검색 자동완성, '기능·화면' 태그면 필수, 이슈·QA는 선택) → 본문 → 등록. 테이블 행 아코디언의 메모 입력도 이 폼 재사용(대상 자동 지정).
- 필터: 태그별 + 미해결/해결(resolved 토글 — 이슈 소통의 완료 처리) + 대상 화면별.
- 상단 요약: "미해결 이슈 3 · QA 1" 카운트.

## 3. 역할명 변경 (요구 3)

첫 진입 선택지·author 저장값·토스트 표기를 **기획자(정주) / 결정권자(대표) / 개발**로 변경. 기존 change_log·메모의 author 값('정주'/'대표'/'게스트')은 표시 시 매핑(정주→기획자(정주), 대표→결정권자(대표), 게스트→개발)으로 소급 처리 — DB 원본은 손대지 않음.

## 4. 미구현 요소 표시 (요구 4)

현재 화면의 FEATURES 중 프로토타입에 `data-func-id` 요소가 없는 행(Later·결정대기 O9 등)도 테이블에 **항상 표시**하되:
- 행 앞에 **[미구현] 뱃지**(gray, micro) + 행 전체 텍스트 tertiary 톤
- 호버해도 프레임 하이라이트 없음(대상 없음 안내 툴팁 "프로토타입 미구현 — IA에만 존재")
- 렌즈 집계(§5)에는 포함 — "IA에는 있으나 화면에 없는 것"이 보이는 게 목적
- 구현 여부 판정은 렌더된 DOM의 `data-func-id` 존재로 동적 판단(하드코딩 목록 금지)

## 5. 마일스톤 집계 (요구 5)

마일스톤 렌즈 바로 아래 **집계 미니 패널**: M1/M2/M3/Later/결정대기 별 기능 수 + 미니 막대(잉크 계열, 오렌지 금지). 토글: **현재 화면 | 전체**. Supabase 실시간 값 기준이라 테이블에서 마일스톤을 바꾸면 집계도 즉시 갱신. 각 항목 클릭 시 테이블이 해당 마일스톤 필터로 전환. 미구현(§4) 건수는 괄호로 병기 — 예: `Later 3 (미구현 3)`.
