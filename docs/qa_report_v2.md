- v2 작업은 전부 `v2` 브랜치(스펙 문서→Supabase 기반→라이브 패널→레이아웃→본 검수). 본 리포트 통과 후 main 병합 + `v2.0.0` 태그.

## DoD-2 · 실시간 동기화 (핵심)

시나리오: 브라우저 B(프리뷰)를 **렌즈 "M1만"**으로 두고, 원격 사용자 "대표"(anon REST)가 `HTM-S7-07`(추천 섹션)의 마일스톤을 M3→M1로 변경.

| 측정 | 결과 |
|---|---|
| B 테이블 드롭다운 값 갱신 | **1013ms** (기준 3초) |
| B 프로토타입 요소 투명도 | 0.25 → **1** (동일 시점, 렌즈 재계산) |
| 토스트 | `대표: HTM-S7-07 마일스톤 M3→M1` (스펙 형식 그대로) |
| M3로 원복 시 | 투명도 0.25 복귀 확인 |

별도 실측(개발 단계): `HTM-S7-15` M2→M3 **779ms** + M 뱃지 갱신. 동작 원리: MsGate가 `data-func-id`로 라이브 마일스톤을 조회하므로 Realtime 수신 즉시 내·원격 모든 브라우저에서 뱃지·잠금 재계산.

## DoD-3 · 하이라이트·고정

- 프로토타입 `HTM-S7-17`(저장 버튼) 호버 → 테이블 행 `is-hot`(주황 워시) + **자동 스크롤로 가시 영역 진입** 실측.
- 행(`HTM-S7-13`) 호버 → 프레임 안 요소 주황 아웃라인(`ia-hot`) ✓, 이탈 시 해제 ✓.
- 행 클릭 = 선택 고정: 다른 행에 호버해도 선택·아코디언 유지, 재클릭 해제 ✓ (v1 스페이스바 대체).

## DoD-4 · 메모·히스토리

- 원격("대표") 메모 insert → **뱃지 "1"** + 토스트 `대표: HTM-S7-13에 메모를 남겼어요`.
- 메모 탭: 최신순 스레드(작성자·시간·ID 칩), 열람 시 뱃지 해제 ✓, ID 칩 클릭 → 테이블 행 점프+선택 ✓.

| V2-F1 | **기능 추가 검증 결함**: 형식 정규식이 `S99` 같은 **미등록 화면 접두사**를 통과시킴 — 검수 중 실제로 쓰레기 행 `HTM-S99-01`이 생성됨. 또한 화면 귀속이 "현재 보고 있는 화면"이라 ID 접두사와 불일치 가능 | `useIaLive.addFeature`: ID 접두사가 `ia_screens`에 실존하는지 검증 추가 + **화면을 ID 접두사에서 자동 유도**. 수정 후 3종 거부 케이스 재실측 통과 |

**부수 기록:** 검수용 데이터가 DB에 남음 — 보관 행 2개(HTM-S7-18, HTM-S99-01), 메모·change_log 다수. 기록 보존 원칙상 삭제 불가하며, 협업 도구의 실사용 이력 예시로 활용 가능.
