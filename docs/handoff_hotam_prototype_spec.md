## 1. 기술 규칙 (그대로 따를 것)

- **스택:** Vite + React + 순수 CSS(변수 기반). 외부 UI 라이브러리 금지. 아이콘은 `lucide-react`만.
- **화면 프레임:** 모바일 390×844. 데스크톱 브라우저에선 **좌측(가로 중앙에서 약간 왼쪽)에 프레임, 우측에 IA 인스펙터 패널(§8)** 배치, 바깥은 `#26422A` 배경. 뷰포트가 좁으면(<1100px) 패널 자동 숨김 + 프레임 중앙 정렬.
- **라우팅:** react-router 없이 자체 상태 라우팅(현재 화면 state + 히스토리 배열로 뒤로가기). 화면 전환 시 200ms 페이드/슬라이드.
- **상태:** React Context 1개(현재 유저, 좋아요/저장/팔로우/찜 토글, 기록 플로우 진행값, 마일스톤 렌즈). localStorage 사용 금지.
- **폰트:** Pretendard CDN(jsdelivr). 숫자는 `font-variant-numeric: tabular-nums`.
- **더미 사진:** 실사 이미지 사용 금지. `gray-100` 면 + 중앙 이미지 라인 아이콘 + 좌하단 12px 라벨(예: "성수 파스타"). 지도는 `gray-100` 면 + 도로 느낌 라인 몇 개 + 핀.
- **이모지 전면 금지.** 마스코트 '탐이'는 점선 원형 플레이스홀더 + "탐이" 라벨로만, **화면당 최대 1회**, 스플래시·온보딩·빈 상태·에러에만.

## 3. 어휘 규칙 (전 화면 강제)

- 선택 버튼(온보딩·기록): **좋았어요 / 그저그래요 / 별로였어요**
- 칩·그룹 헤더·통계·차트: **좋음 / 그저 / 별로**
- 이 두 세트 외 변형 금지('좋아요 그룹', '별로에요' 등). '밴드', 'Elo' 등 내부어는 UI 노출 금지 — 점수 설명 시트에서만 "비교를 통해 계산돼요" 수준으로.
- 카피 톤: 인풋·라벨·버튼=담백, 위트는 빈 상태·에러·온보딩·완료에만("맛집 하나 알려주면 안 잡아먹지" 화법, 협박 소구 금지).

## 5. 더미데이터 스키마 (`data/dummy.js`)

- `users` 10명: id, 닉네임(미식가 느낌 한글), 동네, 아바타 이니셜, 평가 수, 팔로워/팔로잉 수, 인증 여부 1~2명, 한줄 소개
- `restaurants` 24곳: id, 이름, 카테고리(한식/중식/일식/양식/카페/디저트/분식), 동네(성수/연남/을지로/망원), 전체 평점, 리뷰 수, 대표 라벨(플레이스홀더용), [영업 여부, 대표 메뉴·가격 2~3개, 좌석/주차/예약]
- `posts` 30건: id, userId, restaurantId, grade(good/soso/bad), 표시 점수, 본문(1~3문장, 없음도 3건), 사진 수(0~3), 좋아요 수, 댓글 수, 작성 시간, 댓글 배열(2~5개)
- `myList`: good 8곳·soso 5곳·bad 3곳 — **점수는 반드시 밴드 준수**(good 7.0~9.8 / soso 4.0~6.9 / bad 0.5~3.9, 소수 1자리), 그룹 내 점수 내림차순
- `wishes` 6건(게시물 저장 3 + 가게 찜 3), `notifications` 8건(팔로우/좋아요/댓글/랭킹 변동), `ranking` 15명(내 계정 7위), `restaurantRanking` 10곳, `tasteDNA`(카테고리 비중 5개 + 가격대 3구간 + 등급 분포), `searchSuggests`, `regionOptions`
- 내 계정: "정주" — 가본 곳 16 · 위시 6 · 팔로워 42 · 팔로잉 38

## 6. 화면별 명세 — 22화면 (IA 기능ID 기준, M 표기=해당 기능 마일스톤)

각 화면 공통: 정상 상태 + **빈 상태(탐이+위트 카피+CTA) 또는 에러(재시도)** 중 해당되는 것을 실제로 볼 수 있게 — 빈 상태는 데모 스위치(§7)로 전환.

**S13 스플래시** — 탐이 플레이스홀더+로고타입 "hotam". 1.5초 후 A0 자동 이동(M1).
**A0 로그인** — 로고+메인 카피("맛집 하나 알려주면 안 잡아먹지") 1줄+서브 카피 담백 1줄. 버튼 4: 카카오/Google/Apple/이메일로 시작(전부 secondary h48, 브랜드 컬러 로고 점만) + "이메일 회원가입" 텍스트 링크→S14. 아무 버튼이나 누르면 S1로(더미 인증).
**S14 이메일 회원가입** — 4스텝(이메일→인증코드→비밀번호→약관). 상단 진행 표시, 인증코드는 아무 6자리나 통과. 완료→S1.
**S1 온보딩** — 5스텝: ①프로필(이미지 플레이스홀더+닉네임 인풋) ②권한 3종(설명+허용 버튼, 실제 권한 요청 없음) ③지역 선택(M2 — 칩 그리드에서 택1) ④취향 선택(카드 스와이프: 식당 사진 플레이스홀더+이름+동네+종목, 하단 좋았어요/그저그래요/별로였어요 + 가본 적 없어요/건너뛰기, 8장, 진행 "3/8") ⑤완료 "시작하기"→S7. 이전/건너뛰기 상시.
**S7 홈** — 앱바: 로고타입 좌 / 우측 아이콘 3(팔로우추천 M2→S16, 검색→S15, 종→S12). 상단 세그먼트 탭: 팔로잉|추천(M2). 추천 섹션(M3): 가로 스크롤 카드 3장(미식가 1·식당 2). 피드 카드(FeedCard): 아바타+닉네임+시간(→해당 유저 프로필), 등급 칩+점수 뱃지, 본문(→S17), 식당명 링크(→S4), 사진 플레이스홀더(복수 시 가로 스와이프), 액션 행: 좋아요(토글 작동+숫자 증감, 숫자 탭→좋아요 리스트 바텀시트 M2), 댓글(M2 →S17 댓글 영역), 공유(OS 공유 흉내 시트), 저장(토글→위시 반영). 빈 상태: "아직 팔로우가 없어요" + 전체 인기 기록 3개 표시(graceful degrade) + 팔로우 추천 CTA.
**S15 통합검색** — 검색 인풋(placeholder 3종 로테이션: "식당, 미식가 검색"/"오늘은 어느 집을 노려볼까요?"/"어제 그 집 이름이 뭐였더라"), 최근 검색어(M2)·정렬(M2)·필터 칩(M2). 결과 탭: 식당|미식가. 식당 결과→S4, 미식가 결과→프로필. 지도뷰 토글(M3)=지도 플레이스홀더 전환. 결과 없음 상태: "이 집은 아직 아무도 못 찾았어요"+식당 추가 CTA→S18.
**S16 팔로우 추천(M2)** — 리스트: 아바타·닉네임·인증 뱃지·소개·팔로우 버튼(토글 작동).
**S12 알림** — 최신순, 유형별 아이콘(팔로우/좋아요/댓글 M2/랭킹 M2). 각 항목 탭→관련 화면. 빈 상태 탐이.
**S17 게시물 상세(M2)** — 피드 카드 풀 + 댓글 전체(입력창은 눌리면 "프로토타입에선 구경만" 토스트), 미팔로우 작성자면 팔로우 버튼, 반응한 사람들 보기(M3 바텀시트), 하단 식당 요약 카드(M3→S4).
**S9 탐색** — 상단 검색 바(→S15 식당 탭)+필터 칩(M2: 지역/종목/근처 — 탭 시 선택 표시만). 지도 플레이스홀더+핀 6개(핀 탭→S4). 우하단 내 위치 버튼. 리스트 바텀시트: 절반↔풀 두 단계 드래그/탭 전환, 정렬 세그먼트(랭킹순/거리순/최신순 — 실제 재정렬 작동), 행(RestaurantRow): 사진·이름·동네·카테고리·전체 평점·[리뷰 수 M2·거리 M2·찜 토글 M2·예상 점수 M3(점선 뱃지 "예상 8.2")]. 최하단 "여기 없어요? 식당 추가"→S18.
**S18 식당 추가** — 폼: 사진(선택)·식당명·위치·카테고리. 필수 3개 채우면 버튼 활성 → 성공 토스트 "제보 접수! 탐이가 킁킁 맡아볼게요" → 뒤로.
**S4 식당 상세** — 와이드 사진, 이름 20+동네·카테고리, 점수 위계 3단: ①친구 평점(아바타 3개+평균, 탭→친구 평점 리스트 서브 페이지) ②전체 평점 ③큐레이션 점수(M2, '?' 탭→산정 방식 바텀시트, "데이터가 얇으면 전체 평점을 보여드려요" 문구 포함). 리뷰 리스트(=posts 필터, 탭→S17). 찜 토글. [대표 메뉴·가격, 상세 정보(M3) 접이식 섹션]. CTA "방문 기록하기"(주황 1곳)→S2(식당 프리셋).
**S2 기록** — 스텝형: ①가게 선택(검색 인풋=S15 컴포넌트+현 위치 추천 3곳(M2)+없는 식당 추가→S18) ②사진(갤러리 흉내 그리드에서 0~3장 선택) ③본문 인풋(선택) ④만족도: 큰 버튼 3(좋았어요/그저그래요/별로였어요) 택1 필수. 재방문 식당 선택 시 배너 "재방문이네요!"(M2, 이전 평가는 O12 미결로 숨김). 다음→S3.
**S3 비교** — 상단 caption "'좋았어요'끼리 비교 중 · 2/3". A vs B 카드 2장(사진+이름+동네), 중앙 vs, 질문 "어디가 더 좋았어요?". 탭 시 다음 쌍(총 3쌍), 우상단 "건너뛰기". [비교 기준 칩 M3: 맛/가성비/분위기/재방문]. **같은 등급끼리만 비교(교차 만족도 없음 — O9 결정대기 제외).** 완료→S19.
**S19 기록 완료** — 체크 모션+카피 "오늘도 하나 낚아챘어요", 점수 뱃지 크게(예: 8.4, text-brand), '?' 아이콘(M2 산정 방식 시트), 버튼: 내 리스트 보기(주 CTA→S5)/계속 기록(M2→S2)/점수가 마음에 안들어요(M2 — 탭 시 "비교 2번으로 바로잡아요" 시트→S3 재진입 흉내).

## 7. 프로토타입 전용 장치 2개

1. **마일스톤 렌즈:** 프레임 밖 좌측(데스크톱 여백)에 토글 3개 — `M1만` / `M1+M2` / `전체(M3까지)`. M2·M3 기능 요소에는 우상단 micro 뱃지(M2=forest, M3=회청)를 달고, 렌즈 범위 밖 요소는 opacity 0.25+클릭 잠금. 대표가 마일스톤 회의에서 "M1만 보기"로 출시 화면을 확인하는 용도.
2. **데모 스위치:** 렌즈 아래 체크박스 — `빈 상태 보기`(피드·알림·검색결과·S8·별로 그룹이 빈/부족 상태로 전환), `에러 보기`(피드·탐색에 에러+재시도 카드). 상태 4종을 회의에서 바로 시연.

## 8. IA 인스펙터 (우측 패널 — 데스크톱 전용 리뷰 도구)

프로토타입을 쓰면서 IA 문서를 실시간 대조하는 장치. **모든 표시 내용은 `src/data/ia_inspector.js`에서만 읽는다**(하드코딩 금지). 이 파일은 xlsx에서 자동 생성된 것으로 `SCREENS`(27개, 화면ID 키)·`FEATURES`(119개, 기능ID 키)·`IA_META`를 export한다.

**패널 UI** — 프레임 우측 고정 360px, 메모장 느낌: `bg-surface`, `radius-lg`, `shadow-sm`, 상단 헤더 "IA 인스펙터 · v{IA_META.version}"(caption) + 고정 상태 표시. 본문은 라벨(caption, text-secondary)+값(body) 세로 나열, 마일스톤은 렌즈와 동일한 뱃지 스타일, 스크롤 가능.

**동작 규칙**

1. **기능 호버:** 기능이 붙은 모든 컴포넌트에 `data-func-id="HTM-…"` 속성을 단다(§6의 기능ID 그대로 — 이것이 커버리지의 기준). 마우스 호버 시 ①해당 요소에 `border-focus` 2px 아웃라인+살짝 밝은 오버레이로 하이라이트 ②패널에 해당 FEATURES 레코드 표시: **명칭(name) · 화면ID(screen) · 기능ID · 목적/기능 설명(purpose) · UX 카피 노트(copy) · 진입→연결(nav) · 마일스톤 · 중요도(importance) · 이슈/색인(flags) · 비고(note)**. 중첩 요소는 가장 안쪽(구체적인) 기능 우선.
2. **비호버:** 현재 화면의 SCREENS 레코드(화면명·화면ID·목적·마일스톤·카피·비고) + 그 화면 소속 기능 목록(기능ID·명칭·마일스톤 뱃지, 항목 호버 시 프레임 안 해당 요소 역하이라이트).
3. **스페이스바 = 고정 토글:** 누르는 순간 패널 내용을 고정(호버해도 안 바뀜, 헤더에 핀 아이콘+"고정됨"), 다시 누르면 해제. **입력창(input/textarea) 포커스 중에는 무시**(타이핑 방해 금지). 패널 헤더의 핀 아이콘 클릭도 동일 동작.
4. 렌즈·데모 스위치(§7)는 패널 하단으로 이동해 리뷰 도구를 한 곳에 모은다. 터치 기기에선 패널·하이라이트 비활성.

**수정 워크플로(중요):** IA(xlsx) 변경 → `src/data/ia_inspector.js`의 해당 항목 수정 → 저장하면 개발 서버가 즉시 반영(HMR). 파일 상단 주석에 이 규칙 명시. 대량 변경 시엔 Cowork(정주+Claude)에서 xlsx로부터 재생성해 통째로 교체 가능.
